*** 1. Task Spawn 패턴
:PROPERTIES:
:CUSTOM_ID: task-spawn-패턴
:END:
#+begin_src markdown
<task_spawn>
Spawn a **[역할명]** sub-agent using the `Task` tool:
- 구체적 지시사항
- 입력 데이터 명세
- 예상 출력 형식
- 에러 처리 방법
</task_spawn>
#+end_src

*실제 사례*: Unit Tests Bot - Codebase Analyzer → Test Planner → Test Writer → Verifier → Optimizer - 각 단계가 독립적 서브에이전트로 실행 - 병렬 처리 가능한 작업은 동시 실행
