*** Tier 2: Details Layer (Google Workspace)

*역할*: 상세 정보 제공 (Apps Script로 접근)

#+begin_src javascript
// Apps Script: 디테일 제공자
function getProjectDetails(projectName) {
  // 1. Calendar 분석
  var cal = CalendarApp.getDefaultCalendar();
  var events = cal.getEvents(weekStart, weekEnd);
  var projectEvents = events.filter(e =>
    e.getTitle().includes(projectName)
  );

  // 2. Sheets 진행률
  var sheet = SpreadsheetApp.openById('pm-dashboard');
  var row = sheet.getRange('A:A').getValues()
    .findIndex(r => r[0] === projectName);
  var progress = sheet.getRange(row, 2).getValue();

  // 3. Gmail 최근 논의
  var threads = GmailApp.search(
    `subject:${projectName} newer_than:3d`
  );

  // 결과만 리턴 (토큰 절약)
  return {
    hours_this_week: calculateHours(projectEvents),
    progress_percent: progress,
    recent_discussions: threads.length,
    next_deadline: getNextDeadline(projectEvents)
  };
}
#+end_src

*** "무거운 것은 리포로, 가벼운 것만 메모리에"

#+begin_example
무거운 것 (리포로):
  - 상세 문서 (5KB+)
  - 코드
  - 히스토리
  - 디테일한 노트

가벼운 것 (메모리에):
  - 리포 포인터
  - 현재 상태
  - 마지막 동기 시간
  - Workspace 리소스 ID
#+end_example

*** "디테일은 Workspace가, 진실은 리포가"

#+begin_example
Workspace의 역할:
  - 실시간 변동 정보 (진행률, 시간)
  - 협업 정보 (Chat, 댓글)
  - 일정 정보 (Calendar)

Repository의 역할:
  - 영속적 진실 (코드, 문서)
  - 버전 히스토리
  - 완전한 컨텍스트
#+end_example
