*** Tier 2: Details Layer (Google Workspace)

*위치*: Google Sheets, Docs, Calendar, Chat
*접근*: Apps Script로 집계
*역할*: 실시간 디테일 제공
*토큰*: 300 (요약 결과만)

#+begin_src javascript
// Apps Script: 디테일 제공자
function getProjectDetails(projectName) {
  // Google 내부에서 모두 처리
  var hours = getHoursFromCalendar(projectName);
  var progress = getProgressFromSheets(projectName);
  var discussions = getRecentDiscussions(projectName);

  // 요약만 리턴
  return {hours, progress, discussions};
}
#+end_src

** 📊 토큰 효율 (실측 기반)

| 작업 수행 | 8,000 | 600 | **13배** | 리포 직접 |
| **평균** | **7,100** | **440** | **16배** | 실측 평균 |

*** Apps Script 효과가 큰 경우

1. 여러 서비스 조합 (Calendar + Gmail + Sheets)
2. 대량 데이터 집계
3. 주기적 리포트 생성
4. 팀 협업 워크플로우

→ **15-20배** 절감

*** Apps Script 효과가 적은 경우

1. 단순 조회 (Calendar 목록)
2. 단일 리소스 접근
3. 소량 데이터

→ **1-2배** 절감

** 🚀 Apps Script의 진짜 가치

*** 토큰 절약 < 완전 자동화

#+begin_quote
"Apps Script의 진짜 가치는 토큰 절약이 아니라,
Google Workspace에 에이전트를 상주시키는 것이다.

Claude도, 사람도 없이
매주 금요일 17시에 자동으로 리포트를 만들고
팀 Chat에 공유한다.

이것이 진정한 자동화다."
#+end_quote

*** 4대 강력 기능

1. *Bound Scripts*: Sheets/Docs에 로직 내장
2. *Custom Functions*: =CLAUDE_ANALYZE(A1:A10)=
3. *Time-driven Triggers*: 완전 자동 실행
4. *Web App Endpoints*: MCP 양방향 통신

#+begin_src yaml
Tracking:
  - "어디에 무엇이"

#+begin_quote
"1 Repository = 1 Tracking File"

리포가 늘어도 추적 파일만 늘어남.
리포 내부 복잡도는 추적 파일에 영향 없음.
#+end_quote

*** 3. Repository as Source of Truth

#+begin_src yaml
진실의 우선순위:
  1. Git Repository (코드, 문서)
  2. Workspace (실시간 협업 데이터)
  3. Tracking (포인터만)

동기화 방향:
  Repo → Workspace (Apps Script 자동)
  Workspace → Repo (수동 또는 승인)
  Tracking ← Both (상태만 반영)
#+end_src
