* 개요

agent-shell 패키지와 ACP/A2A 프로토콜을 활용하여 Emacs를 중심으로 OpenCode, Claude Code, Goose 등 다양한 AI 코딩 에이전트를 통합하는 전략입니다.

** 핵심 아이디어
- **단일 인터페이스**: Emacs agent-shell
- **표준 프로토콜**: ACP → A2A (Linux Foundation)
- **다중 에이전트**: OpenCode, Claude Code, Goose, Gemini CLI, Cursor 등
- **비용 최적화**: OpenCode Zen의 큐레이티드 모델

* agent-shell: Emacs 네이티브 에이전트 셸

** 개요

- **패키지**: MELPA 제공
- **라이선스**: 오픈소스

* ACP/A2A 프로토콜

** ACP (Agent Communication Protocol)

*** 개요
- **개발**: IBM Bee 팀
- **출시**: 2025년 3월
- **목적**: AI 에이전트, 애플리케이션, 인간 간 통신 표준화

*** 기술적 특징

**** REST 기반 구조
- 단순하고 잘 정의된 REST 엔드포인트
- 표준 HTTP 패턴
- JSON-RPC 같은 특수 방식 불필요

**** SDK 불필요
#+begin_src bash
# curl로 직접 호출 가능
curl -X POST https://agent.example.com/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello"}'
#+end_src

- curl, Postman, 브라우저로 접근 가능
- Python, TypeScript SDK 제공 (선택사항)

**** 비동기 우선 설계
- 장시간 실행 작업에 최적
- 동기 요청도 지원
- 스트리밍 상호작용 가능

**** 멀티모달 지원
- MIME 타입 활용
- 텍스트, 이미지, 오디오, 비디오
- 모든 데이터 형식 지원

** A2A (Agent-to-Agent Protocol)

*** 개요
- **개발**: Google
- **출시**: 2025년 4월
- **관리**: Linux Foundation
- **참여**: 100개 이상 기술 기업

*** 출범 배경
- 엔터프라이즈 환경에서 AI 에이전트 확장 문제 해결
- 플랫폼, 벤더, 프레임워크 무관하게 상호운용
- 중립적 거버넌스 (Linux Foundation)

*** ACP와의 통합

**** 타임라인
- 2025년 3월: IBM, ACP 출시
- 2025년 4월: Google, A2A 출시
- 2025년 6월: ACP가 A2A에 병합

**** 통합 이유
- 접근 방식의 유사성
- 표준화의 중요성
- 중복 노력 방지
- 커뮤니티 통합

**** 마이그레이션
- ACP 팀이 A2A에 기술 및 전문성 기여
- 마이그레이션 경로 제공
- 문서화된 전환 가이드
- 원활한 이전 지원

*** A2A의 이점

**** 광범위한 지원
- 100개 이상 기업 참여
- Google의 리더십
- Linux Foundation 중립성
- 산업 표준으로 부상

**** 엔터프라이즈 최적화
- 확장성 검증
- 보안 강화
- 프로덕션 준비
- 장기 지원 보장

** MCP vs ACP/A2A 비교

| 특성 | MCP | ACP/A2A |
|------|-----|---------|
| **개발자** | Anthropic | IBM → Google + Linux Foundation |
| **초점** | 모델-컨텍스트 연결 | 에이전트-에이전트 통신 |
| **통신** | JSON-RPC | REST API |
| **SDK 필요** | 예 | 아니오 (선택사항) |
| **비동기** | 지원 | 기본값 |
| **거버넌스** | Anthropic | Linux Foundation (중립) |
| **채택** | Claude 생태계 | 100+ 기업 |

*** 상호보완성
- **MCP**: 도구 호출, 컨텍스트 제공
- **A2A**: 에이전트 간 통신, 조율

두 프로토콜은 함께 사용 가능하며, 서로 다른 계층에서 작동합니다.

* OpenCode Zen: 큐레이티드 모델 게이트웨이

** 개요

OpenCode Zen은 **"AI 코딩 에이전트용으로 테스트되고 검증된 모델"**을 제공하는 게이트웨이 서비스입니다.

- **상태**: 베타
- **제공**: 선택사항 (필수 아님)
- **모델 수**: 15개 큐레이션
- **가격**: Pay-as-you-go

** 왜 OpenCode Zen인가?

*** 1. 검증된 모델만 제공
"코드 생성과 도구 호출에 우수한 성능을 보이는 모델"만 선별
- 품질 보장
- 시행착오 감소
- 프로덕션 준비

*** 2. 투명한 가격 정책
#+begin_quote
"OpenCode Zen passes along any price drops by selling at cost,
the only markup is to cover processing fees"
#+end_quote

- **원가 판매**: 가격 인하 즉시 반영
- **마크업 최소화**: 처리 수수료만
- **카드 수수료**: 원가 그대로 전가
- **추가 요금 없음**

*** 3. 무료 모델 제공
- Grok Code Fast 1 (현재 무료)
- Big Pickle (베타 무료)
- GPT 5 Nano
- Code Supernova (실험적, 베타 무료)

*** 4. 팀 관리 기능
- 역할 기반 접근 (Admin/Member)
- 멤버별 월간 지출 한도
- 팀별 모델 제한
- 개인 API 키 사용 옵션

*** 5. 프라이버시 보장
- **미국 호스팅**: 모든 모델
- **제로 보존 정책**: 대부분 모델이 데이터를 학습에 미사용
- **예외**: Grok/Big Pickle (개선용 사용 가능)
