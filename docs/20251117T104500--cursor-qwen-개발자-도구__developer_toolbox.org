* 개요

Cursor CLI와 Qwen Code는 agent-shell과 함께 사용할 수 있는 터미널 기반 AI 코딩 도구입니다.
각각 고유한 특징과 강점을 가지고 있으며, ACP/A2A 프로토콜을 통해 Emacs에서 통합 가능합니다.

** 비교 요약

| 항목 | Cursor CLI | Qwen Code |
|------|-----------|-----------|
| **개발사** | Cursor (Anysphere) | Alibaba Qwen Team |
| **출시** | 2025년 8월 (베타) | 2025년 7월 |
| **기반** | Claude Code 대안 | Gemini CLI 포크 |
| **주 모델** | GPT-5, Claude Sonnet | Qwen3-Coder 480B |
| **무료 옵션** | IDE 플랜 필요 | 2,000 요청/일 (OAuth) |
| **ACP 지원** | cursor-acp 어댑터 | @qwen-code/qwen-code |
| **특화** | IDE 통합, 다중 모델 | 대규모 코드베이스, 에이전트 |
| **컨텍스트** | IDE 설정 재사용 | 256K-1M 토큰 |

* Cursor CLI

** 개요

Cursor CLI는 "AI coding agent you interact with via the terminal"로 설명되는 도구입니다.
2025년 8월 7일 베타 출시되었으며, Cursor IDE의 터미널 확장판입니다.

- **공식 사이트**: https://cursor.com/cli
- **상태**: 베타 (활발히 개발 중)
- **라이선스**: 독점 (Cursor 플랜 필요)

** Cursor IDE vs Cursor CLI

*** 핵심 차이점

**** 인터페이스
- **IDE**: VS Code 기반 완전한 IDE
  - 시각적 코드 편집
  - 인덱싱 및 컨텍스트 검색 강력
  - 빠른 네비게이션

- **CLI**: 터미널 네이티브
  - 명령줄 중심 워크플로우
  - 원격 서버에서 사용 가능
  - 컨텍스트 전환 없음

**** 사용 사례
- **IDE**:
  - 일상적 코드 편집
  - 프로젝트 전체 리팩토링
  - 시각적 디버깅

- **CLI**:
  - 터미널 기반 워크플로우
  - 원격 서버 작업
  - CI/CD 파이프라인
  - 스크립팅 자동화

**** 설정 공유
- **공통점**: 동일한 보안 정책 (ask-to-apply)
- **재사용**: IDE 설정을 CLI가 활용
- **멘탈 모델**: 동일 (채팅 → 승인 → 실행)

*** 장점

**** 1. 다중 모델 지원
슬래시 명령으로 즉시 전환:
- GPT-5, GPT-5 Codex
- Claude Sonnet 4.5, 4
- Gemini 2.5 Pro
- 기타 Cursor 지원 모델

명령어:
#+begin_src bash
/model
#+end_src

**** 2. 명령어 생성 (Ctrl+K)
터미널에서 Ctrl+K 누르면:
1. 프롬프트 바 열림
2. 원하는 작업 설명
3. AI가 명령어 생성
4. 승인 후 실행

예시:
#+begin_src
Ctrl+K > "find all large files over 100MB"
→ find . -type f -size +100M -exec ls -lh {} \;
#+end_src

**** 3. 배경 에이전트
- 장시간 작업을 백그라운드로
- 터미널 블로킹 방지
- 다중 작업 병렬 실행

**** 4. MCP 통합
- Model Context Protocol 지원
- 외부 도구, 데이터 소스 연결
- 더 많은 정보 기반 응답

**** 5. 크로스 플랫폼
- VS Code
- JetBrains
- Android Studio
- 모든 터미널 환경

*** 단점 (베타 한계)

**** 1. MCP 서버 호환성 부족
현재 문제:
- Obsidian 노트 조회 실패
- GitHub 이슈 생성 실패
- IDE에서는 작동하는 MCP가 CLI에서 안 됨

원인: 베타 단계로 안정성 부족

**** 2. 컨텍스트 첨부 미흡
IDE의 ~@~ 기호 기능:
- ~@file~: 파일 첨부
- ~@folder~: 폴더 첨부

CLI에서는 동작이 다름:
- 동일한 방식으로 작동 안 함
- 수동 컨텍스트 제공 필요

**** 3. 기능 제한
현재 미지원:
- 커스텀 슬래시 명령
- ~/hooks~ 같은 고급 기능
- 일부 IDE 전용 기능

**** 4. 비용
- Cursor 플랜 필요 (무료 아님)
- IDE 라이선스 연동

*** ACP 통합: cursor-acp

Cursor CLI는 기본적으로 ACP를 지원하지 않지만, 커뮤니티 어댑터가 있습니다.

**** cursor-acp 어댑터

**개발**: roshan-c
**GitHub**: https://github.com/roshan-c/cursor-acp
**용도**: Cursor CLI를 ACP로 브리징

**주요 기능**:
- Cursor ↔ ACP 실시간 스트리밍
- 도구 호출 매핑:
  - grep/glob 검색
  - bash 실행
  - 파일 읽기/쓰기
- default/plan 모드 지원
- 취소 처리
- 인증 안내

**설치**:
#+begin_src bash
npm install -g cursor-acp
#+end_src

**사용**:
#+begin_src bash
cursor-acp
#+end_src

**Zed 통합**:
설정 파일에 "agent_servers"로 등록

**환경 변수**:
- ~CURSOR_AGENT_EXECUTABLE~: 특정 바이너리 지정

**인증**:
#+begin_src bash
cursor-agent login
#+end_src

**** agent-shell에서 사용

agent-shell 문서에서 언급된 설치 방법:
#+begin_src bash

#+end_src

**주의**:
- 실제로는 roshan-c/cursor-acp가 메인 프로젝트
- @blowmage 버전은 확인 불가
- roshan-c 버전 사용 권장

* Qwen Code

** 개요

Qwen Code는 Alibaba Qwen Team이 개발한 "AI-powered command-line workflow tool"입니다.
Qwen3-Coder 모델에 최적화되어 있으며, Gemini CLI를 기반으로 개발되었습니다.

- **GitHub**: https://github.com/QwenLM/qwen-code
- **출시**: 2025년 7월
- **라이선스**: Apache 2.0 (완전 오픈소스)
- **모델**: Qwen3-Coder 480B-A35B-Instruct

** Qwen3-Coder 모델

*** 사양

**** 아키텍처
- **파라미터**: 480B (총), 35B (활성)
- **타입**: Mixture-of-Experts (MoE)
- **컨텍스트**: 256K 토큰 (기본), 1M 토큰 (YaRN 확장)

**** 학습 규모
- **토큰**: 7.5조 (70% 코드)
- **목표**: 코딩 능력 + 일반/수학 능력 균형

*** 핵심 혁신

**** 1. Agent RL (장기 강화학습)
**전통적 RL**:
- 단일 턴 작업
- 즉시 보상

**Agent RL**:
- 다중 턴 상호작용
- 환경 도구 사용
- 실제 작업 해결

**확장성**:
- 20,000개 독립 환경 병렬 실행
- 대규모 학습 가능

**** 2. Execution-Driven RL
코드 실행 결과로 학습:
- 정적 분석이 아닌 동적 실행
- 실제 작동 여부 확인
- 광범위한 실제 작업 적용

**** 3. 합성 데이터
Qwen2.5-Coder로 데이터 품질 향상:
- 고품질 학습 데이터 생성
- 다양한 코딩 시나리오

*** 성능

**** SWE-Bench Verified
"State-of-the-art performance among open-source models"
- 테스트 시간 스케일링 없이 달성
- Claude Sonnet 4와 비교 가능

**** 에이전트 능력
다음 영역에서 Claude Sonnet 4 수준:
1. **Agentic Coding**: 자율적 코드 작성
2. **Agentic Browser-Use**: 웹 브라우징 자동화
3. **Agentic Tool-Use**: 도구 사용 능력

**** Terminal-Bench
Qwen Code CLI 정확도:
- Qwen3-Coder-480A35: **37.5%**
- Qwen3-Coder-30BA3B: **31.3%**

** Qwen Code CLI 기능

*** 핵심 기능

**** 1. 대규모 코드베이스 처리
전통적 컨텍스트 윈도우 초과:
- 256K-1M 토큰 활용
- 저장소 전체 분석
- 동적 데이터 처리 (PR 등)

예시:

#+end_src

**** 2. 워크플로우 자동화
복잡한 운영 작업:
- Pull Request 처리
- 복잡한 리베이스
- 의존성 업데이트
- 코드 마이그레이션

**** 3. 향상된 파서
Qwen-Coder 모델 특화:
- 최적화된 파싱
- 더 나은 코드 이해
- 정확한 수정 제안

**** 4. 비전 모델 지원
멀티모달 기능:
- 이미지 자동 감지
- 비전 모델로 자동 전환
- 스크린샷, 다이어그램 분석
