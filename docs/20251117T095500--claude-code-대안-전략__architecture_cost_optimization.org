* 문제 상황

** 현재 문제점
- Claude Code Max 플랜 사용 ($220/월)
- Weekly limit에 걸려 며칠간 사용 불가
- 높은 비용에도 불구하고 제한적
- 특정 벤더(Anthropic)에 종속적

** 요구사항
1. 비용 효율적인 대안
2. Weekly/daily limit이 없는 무제한 또는 준무제한 옵션
3. 특정 벤더 종속 탈피
4. 에이전트-투-에이전트(A2A) 워크플로우 구축 가능
5. MCP 프로토콜 중심의 통합
6. Emacs 통합

* 핵심 인사이트: 정액제의 한계

** AI 업계의 현실
현재 AI 산업에서 **진정한 의미의 무제한 정액제는 존재하지 않습니다**.

이유:
- 사용자들이 $20-$200/월 구독료로 감당할 수 없는 수준의 토큰을 소비
- Claude Code도 $220/월에 weekly limit 적용
- AI 회사들은 무제한처럼 보이는 플랜을 제공하고 싶지만, 경제성 때문에 불가능

** 현실적 대안 전략
1. **초저비용 API**: DeepSeek (Claude 대비 1/27 가격)
2. **관대한 무료 티어**: Groq, Google Gemini
3. **완전 로컬**: Ollama (하드웨어만 필요, 무제한)
4. **하이브리드**: 작업별로 다른 모델 조합

* 비용 효율적 AI API 옵션

** 1. DeepSeek: 초저비용 + 실질적 무제한

*** 가격
| 항목 | DeepSeek | Claude | 비율 |
|------|----------|--------|------|
| Input | $0.55/1M | $15/1M | 1/27 |
| Output | $2.19/1M | $60/1M | 1/27 |

- **Claude 대비 27배 저렴** (비용의 3.6%)
- Anthropic Claude 2.1 대비: Input 14배, Output 10배 저렴

*** Rate Limits의 혁신
DeepSeek의 독특한 접근:
- **전통적인 rate limit 없음**
- 요청 거부 대신 **응답 속도 조절**
- 트래픽 과다 시 느려지지만 계속 작동
- Rate-limited 요청에 대해 **과금하지 않음**

*** 비용 절감 기능
- **Context caching**: 캐시된 데이터에 대해 74% 비용 절감
- **Off-peak 할인**: 16:30-00:30 UTC에 50-75% 할인
- **무료 크레딧**: 신규 계정에 5백만 토큰 ($8.40 상당)
- 활동 유지 시 매월 갱신

*** 성능
- **최고 속도**: 34 tokens/second (업계 최고)
- Fireworks: 24 t/s
- Hyperbolic: 23 t/s
- OpenRouter: 0.40-0.45s first-token latency

*** 결론
**DeepSeek는 실질적으로 "무제한에 가까운" 가장 현실적인 옵션**
- 토큰당 비용이 너무 저렴해서 대량 사용 가능
- Rate limit이 없어 속도만 감수하면 무한정 사용
- A2A 워크플로우에 최적

** 2. Groq: 빠른 무료 티어

*** 가격
- **무료 티어**: 관대한 제공
- **유료 티어**: 개발자, 엔터프라이즈

*** Rate Limits (무료 티어)
| 모델 | RPM | TPM | 일일 제한 |
|------|-----|-----|---------|
| llama3-8b-8192 | 30 | 30,000 | 14,400 |
| llama-3.1-8b-instant | - | 6,000 | - |

- Organization 레벨에서 적용
- 모델별로 다름
- Console에서 실시간 확인 가능

*** 대표 모델 가격 (유료)
| 모델 | Input | Output | 속도 |
|------|-------|--------|------|
| Llama 3.1 8B | $0.05/M | $0.08/M | 840 TPS |
| GPT OSS 20B | $0.075/M | $0.30/M | 1,000 TPS |
| Llama 3.3 70B | $0.59/M | $0.79/M | 394 TPS |

*** 특징
- Batch API로 50% 비용 절감
- Prompt caching 지원 (캐시 히트 시 50% 할인)
- 숨겨진 비용 없음
- 선형적이고 예측 가능한 가격

*** 적합성
- 실험 및 프로토타이핑
- 경량 작업
- 무료 티어로 시작 가능

** 3. Ollama: 완전 로컬 무제한

*** 개요
- **완전 로컬 실행**
- **진정한 무제한** (하드웨어만 필요)
- 프라이버시 100% 보장
- 인터넷 불필요

*** MCP 통합
2025년 현재 Ollama와 MCP 통합이 활발:

**** 주요 프로젝트
- **ollmcp** (MCP Client for Ollama): 터미널 TUI
  - 다중 MCP 서버 연결
  - STDIO, SSE, HTTP 지원
  - Ollama Cloud 모델 지원

- **Ollama MCP Bridge**:
  - 파일시스템 조작
  - 웹 검색
  - 이메일/문서 관리
  - 코드/GitHub 상호작용
  - 이미지 생성
  - 영구 메모리

- **Dive**: MCP 호스트 앱
  - 로컬 LLM이 MCP 도구 호출 가능

**** Tool Calling 지원 모델
- Qwen2.5-7B-Instruct
- Llama3
- 기타 Ollama의 tool-compatible 모델들

*** 장점
- **비용 제로** (하드웨어만)
- **데이터가 로컬에만 존재**
- **진정한 무제한**
- MCP 서버와 완전 호환

*** 단점
- 하드웨어 요구사항 (GPU 권장)
- 클라우드 모델 대비 낮은 성능
- 초기 설정 복잡도

*** 적합성
- 민감한 데이터 작업
- 오프라인 환경
- 프라이버시 중요 프로젝트
- 실험 및 학습

** 4. OpenRouter: 유연한 멀티 모델 게이트웨이

*** 개요

*** 가격 정책
- **플랫폼 수수료**:
  - 카드: 5.5% (최소 $0.80)
  - 암호화폐: 5% (최소 없음)
- **BYOK (Bring Your Own Key)**:
  - 자신의 API 키 사용
  - 5% 사용료
  - 향후 정액 월구독 전환 예정

- **크레딧 만료**: 365일

*** 지원 모델 예시

- DeepSeek

*** 장점

- 비용-성능 트레이드오프 최적화
- 빠른 프로토타이핑
- 무료 모델 접근

*** 언제 사용?

* MCP (Model Context Protocol)

** 개요
- Anthropic이 개발한 **AI와 외부 도구 통합의 표준 프로토콜**
- 에이전트 간 연결의 핵심
- 오픈 프로토콜

** 주요 특징
- 자연어를 다양한 도구 및 서비스의 액션으로 변환
- 도구 호출 기능을 가진 모든 LLM과 호환
- 표준화된 인터페이스

** Emacs 통합

*** mcp.el
- **메인 MCP 클라이언트**
- MELPA 제공
- MCP 서버 관리 및 연결
- Hub 인터페이스 (mcp-hub)
- 자동 서버 시작

*** gptel 통합
- gptel에서 MCP 지원
- mcp.el과 연동
- 도구 호출 관리

*** org-mcp
- **Org-mode MCP 서버**
- AI가 Org 파일과 상호작용
- 구조화된 API 제공

*** emacs-mcp
- Emacs 환경/설정 정보를 LLM에 제공
- 패키지, 함수, 키바인딩, 문서 접근

*** 참고 자료
- 블로그: [[https://blog.kaorubb.org/en/posts/gpt-mcp-setup/][A Refined Emacs LLM Environment with gpt.el & mcp.el]]
- GitHub Discussion: [[https://github.com/ahyatt/llm/discussions/117][Generic mcp.el for Emacs]]

*** 지원 MCP 서버
- Filesystem
- Fetch
- GitHub
- DuckDuckGo
- 기타 다수

** MCP의 전략적 중요성
- **벤더 중립적**: 특정 AI 회사에 종속되지 않음
- **에이전트 간 연결**: A2A 워크플로우의 표준
- **Emacs 중심**: mcp.el로 Emacs를 허브로 사용
- **확장 가능**: 다양한 도구 추가 용이

** A2A (Agent-to-Agent) 연결

#+begin_example

  ↓ MCP Protocol

  ↓ MCP Protocol

*** 동일 인터페이스의 장점
- 에이전트 교체 용이

- 벤더 종속 없음

*** Emacs의 역할

- **Hub**: MCP 서버 중앙 관리
- **Interface**: 사용자 인터페이스 (Org-mode)
- **Memory**: 작업 히스토리 및 컨텍스트
